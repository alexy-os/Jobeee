Представленный источник описывает архитектуру и принципы платформы Jobeee.net, которая функционирует как социальная обучающая сеть, специализирующаяся на симуляции реальных рабочих процессов. Ключевая миссия платформы — позволить участникам развивать hard- и soft-навыки в игровых, мультикультурных форматах, при этом строго соблюдается правило «нет найма на площадке», что снимает юридические и этические риски. Центральными элементами архитектуры являются Company Studio для создания симуляций компаниями, Assessment Layer для оценки навыков через артефакты и инновационный Vent / Reframe Module, который обучает конструктивному управлению негативными эмоциями. Для повышения доверия и вовлечения платформа использует модель «Private Play» (анонимное прохождение) с возможностью последующей добровольной верификации и сертификации навыков, что гарантирует высокий уровень контроля пользователя над своими данными и прогрессом.

Понял ограничения — **это социальная обучающая платформа**, не рекрут-маркетплейс; **никаких прямых откликов и трудоустройства** на платформе. Все фичи и правила проектируются вокруг обучения реальной работе, культурной адаптации и безопасного управления эмоциями (включая «хейт»), при этом сохраняется международный формат и отсутствие локальных каламбуров.
Ниже — концентрированное, практическое руководство: принципы, набор ключевых функций, архитектура симуляции (на примере кофейни), модуль «контролируемого выпуска пара» (Vent / Hate-handling), правила модерации, верификация компаний, метрики успеха, приватность и краткий набор мини-челленджей для студента-филолога. # Миссия (одно предложение)
jobeee.net — обучающая мультикультурная платформа-соцсеть, где организации создают реальные рабочие симуляции, а участники прокачивают hard- и soft-навыки в игровых форматах без прямого найма.
# Базовые принципы платформы
1. **Нет найма на площадке.** Платформа не публикует вакансий и не принимает отклики — только обучение/симуляции/профили-карточки навыков.
2. **Company-owned simulations.** Симуляции создаёт компания (или локальный бизнес) — передаёт атмосферу, правила, дресскод и кейсы.
3. **Play-to-learn, proof-based.** Навык подтверждается артефактами (видео, скриншоты, peer review), не текстовыми заявлениями.
4. **Этичный Vent Mode.** Пользователь может «выпустить пар» в контролируемом формате с автоматической «цивилизующей» переработкой текста/речи.
5. **Мультипрофильность.** Пользователь собирает профиль из блоков (Teaching, Research, Job-Sim, Social). Нет одного «единого» шаблона.
6. **Культурная адаптивность.** Виджеты и шаблоны локализуются (UX для Японии ≠ UX для Бразилии), но формат и правила одинаковы глобально.
# Ключевые функции (MVP-ориентированные, удобные для обсуждения с командой)
* **Company Studio (Builder).** Инструмент для создания симуляций: сценарии, роли, чек-листы, видео-локации, правила дресскода, expected outcomes, оценочные метрики.
* **Simulation Player.** Интерфейс участника: инструкции, таймер, интерактивные действия, запись выполнения, submit артефактов.
* **Assessment Layer.** Автоматизированная оценка (метрики) + peer review + опциональная модерация человеком.
* **Vent / Reframe Module.** Структурированный шаблон для «выпуска пара» + AI-помощник, который генерирует конструктивные ответы/альтернативы.
* **Multiblock Profile (Widgets).** Drag-and-drop виджеты: Short Bio, Skill Badges, Simulations, Teaching Modules, Micro-portfolio.
* **Creator Marketplace.** Креаторы (университеты, компании, независимые авторы) публикуют симы/виджеты; платные/бесплатные.
* **Reputation & Trust.** Рейтинги создателей симуляций, badges для участников, верификация компаний.
* **Localization Toolkit.** Шаблоны культурной адаптации, переводчики и локальные кейсы.
* **Privacy & Consent Controls.** Разрешения на запись, возможность анонимного участия в отработке конфликтов.
# Пример симуляции: «Кофейня в маленьком городе» — подробный сценарий
Цель: дать кандидату-баристе/администратору/официанту локальный опыт работы, оценить hard (приготовление напитков, POS) + soft (коммуникация, стресс).
**Компоненты симуляции**
1. **Описание роли** — роль (Barista), дресскод, длительность смены (симуляция 20–45 минут), KPI (скорость, точность заказа, вежливость).
2. **Сценарии** — набор мини-заданий: готовка стандартного латте; rush-hour (3 заказа одновременно); жалоба клиента; кассовая ошибка; смена при закрытии.
3. **Интерактивные элементы** — меню, POS-интерфейс, таймер, чек-лист по гигиене.
4. **Артефакты** — короткое видео процесса (или screen recording), заполненный чек-лист, короткий self-reflection (1–2 предложения).
5. **Оценка** — автоматические метрики (время заказа, совпадение рецепта) + 2 peer оценки (отрецензировано сотрудниками кафе/университетскими менторами).
6. **Learning outcomes** — hard: техника приготовления, POS-логика; soft: управление очередью, клиентская эмпатия, стресс-менеджмент.
**Игровые уровни**
* Level 1: базовые напитки — 1 задача (30 мин).
* Level 2: rush-hour — multi-tasking (45 мин).
* Level 3: конфликт с клиентом — roleplay (моделируемый диалог, 15–20 мин).
Каждый уровень даёт badge/микро-сертификат.
**Flow для участника**
1. Read brief → Accept role → Start timer.
2. Perform tasks → Record short proof (video/audio) → Submit artifacts + self-reflection.
3. Receive automated metrics + 2 peer reviews → Get feedback + suggested micro-lessons.
# Vent / Hate-handling: как научить «реагировать красиво»
Цель: дать безопасный формат выражения эмоций и обучение конструктивной реакции без эскалации.
**Структура Vent Mode (шаблон подачи эмоции)**
1. **Context** (что случилось — 1 предложение).
2. **Emotion** (как я себя чувствую — 1 слово).
3. **Impact** (как это повлияло — 1 предложение).
4. **Desired outcome** (что хочу — 1 предложение).
5. **Request** (что прошу платформу/сообщество — 1 действие).
**AI-reframe pipeline (автоматический)**
* Шаг 1: принимает Vent (как есть).
* Шаг 2: предлагает 3 альтернативных формулировки:
* *Empathic professional* (нейтрально-профессиональная),
* *Firm constructive* (чётко и без хамства),
* *Public learning post* (трансформирует в мини-case для обучения сообщества).
* Шаг 3: пользователь выбирает и публикует либо сохраняет черновик.
* Шаг 4: если сообщение потенциально нарушает правила — показывается мягкая блокировка: «Мы поможем выразить мощно и корректно — вот варианты».
**Sandbox «Разнести хейтера» (контролируемая игра)**
* Участник может создать аватар и «виртуальную команду» ИИ-партий, чтобы проиграть сцену.
* Результат — не публичная травля, а обширный фидбэк: какие приёмы были эффективны, какие под действиям нарушали бы этику.
* Публичные записи из sandbox доступны только с согласия (для обучения).
**Примеры полезных ответов (ready templates)**
* *Низкая тяжесть (раздражение)*: «Я понимаю ваше раздражение; спасибо, что поделились. Могу ли я уточнить одно? Это поможет понять ситуацию точнее.»
* *Средняя (оскорбление)*: «Мне неприятно слышать это в наш адрес. Давайте сфокусируемся на фактах: что произошло и какого результата вы ожидаете?»
* *Высокая (агрессия)*: «Ваш тон нарушает правила платформы. Если у вас есть конкретные факты — изложите их без оскорблений; в противном случае мы предложим медиатора.»
# Модерация и доверие
* **Авто-детекция**: AI анализирует тон, токсичность и вероятность фальсификации артефактов.
* **Community reporting**: жалобы → временная блокировка публикации → human review.
* **Reputation decay**: повторные нарушения → понижение репутации, доступ к sandbox/созданию симуляций ограничен.
* **Transparent appeals**: механика апелляций с логом действий.
# Company onboarding / требования к симуляции
* Подтверждение юридического лица / идентификация ответственного контактного лица.
* Стандартный набор полей: цель симуляции, список разрешённых записей (камеры/микрофон), конфиденциальность данных, expected outcomes, чек-листы оценивания.
* Правила по культурным нормам и дресскоду: компании обязаны описать эти правила и примеры допустимого поведения.
* Обязательная секция «Этика и безопасность»: какие реакции разрешены/запрещены, правила обработки жалоб.
# Метрики успеха (какие KPI отслеживать)
* % завершивших симуляцию (completion rate).
* Изменение навыков (pre/post self-assessment + peer scores).
* Retention (повторные симуляции / публикации).
* Quality of artifacts (peer score average).
* Ratio положительных реформулировок в Vent Mode (сколько пользователей превратили негатив в конструктив).
# Приватность и юридика (важные принципы)
* Явное согласие на запись/публикацию видео.
* Возможность анонимного участия в справочных сценариях.
* Данные хранятся шифрованно; компании не получают персональных данных напрямую (только агрегированные результаты/артефакты с согласия).
* Политика «no hiring»: платформа в явной форме запрещает трансакции найма, чтобы избежать юридических и этических проблем.
# Мини-челленджи для студента-филолога (30–60 минут каждый)
1. **Редактирование меню кафе (30 мин)** — отдать меню с 6 пунктами, цель: сделать текст понятным, дружелюбным и локализованным; сдать правки + краткая justification строка.
2. **Клиент-жалоба (30–40 мин)** — симулированный диалог: клиент недоволен напитком; подготовить 2 варианта ответа: эмпатичный + конструктивный; записать 1-мин видео-репетицию.
3. **Короткий лонгрид «О нашей кофейне» (45–60 мин)** — написать 200–300 слов для локальной аудитории; оценка по читабельности и стилю.
4. **Микро-инструктаж (30 мин)** — создать 3-шаговое руководство для нового баристы (чек-лист); цель: максимальная ясность.
5. **Roleplay: rush hour (45 мин)** — прописать свои реплики при нагрузке, затем проиграть с AI-коучем 5 минут.
6. **Локализация вывески (30 мин)** — адаптировать простую вывеску/слоган на 2 локальных языка (краткая justification культурных решений).

# Коротко — идея в одном предложении
Анонимная «песочница» + опциональная сертификация: игроки могут без следов пробовать любые симы; только сознательное, повторное и верифицированное прохождение превращается в навык в профиле.
# Что это решает (практически)
* Снимает страх экспериментировать (ничего не «пломбируется» в профиле автоматически).
* Увеличивает вовлечение — люди пробуют больше симов.
* Даёт чёткий путь «игра → навык»: только сознательное подтверждение и верификация добавляют навык в профиль, повышая доверие работодателей/компаниям и самому игроку.
# Принципы приватности и телеметрии
1. **Эфемерные сессии по умолчанию (Private Play).**
* По умолчанию симуляции запускаются в «анонимной» сессии: никакие данные о действиях не привязываются к постоянному профилю.
* Сессия может хранить временные локальные артефакты (локально в браузере/устройстве) или в шифрованном виде на сервере с коротким TTL и только с согласия пользователя.
2. **Ясное информированное согласие при сохранении.**
* Если пользователь хочет «сохранить прогресс» или «пометить для портфолио», показываем явный диалог: что именно сохраняется, как долго и для каких целей.
3. **Телеметрия — только по Opt-in.**
* Автоматические метрики (время, маршруты кликов и т.п.) собираются **только** при явном согласии для цели улучшения симуляции и/или для прохождения сертификации.
4. **Отсутствие скрытой привязки.**
* Даже если пользователь вернулся в ту же симуляцию с тем же устройством — пока он не авторизовался/не сохранил результаты, платформа не делает permanent link между этими сессиями и его профилем.
# Путь пользователя: от анонимной игры до сертифицированного навыка (пошагово)
1. **Запуск (анонимно).** Пользователь кликает «Play demo» — запускается Private Play. Нет следов в профиле.
2. **Играть сколько хочешь.** Все результаты — временные. Можно переключиться на другие симы, выходить, возвращаться.
3. **Метка «хочу сохранить» (bookmark).** Пользователь кликает «Save run» → появляется диалог с опциями: «Save locally», «Save to drafts (private)», «Save & Submit for Peer Review».
4. **Draft → Review.** Если сохраняет в «черновики», он может позже отправить run на peer-review или на company-review.
5. **Требования к сертификации.** Чтобы претендовать на сертификацию, нужно:
* выполнить симуляцию N раз (настраиваемо; например, 3 успешных прохождения),
* собрать M артефактов (видео, чек-лист, self-reflection),
* иметь минимальный avg peer review score ≥ X,
* пройти финальный real-world test / assessment от компании (встроенный в workflow).
6. **Подать заявку на сертификацию.** После выполнения требований пользователь инициирует «Certification request». Здесь он даёт согласие на передачу необходимых данных компанию-верификатору и на сбор ограничённой телеметрии для проверки подлинности.
7. **Верификация.** Компания/модератор просматривает материалы и/или назначает live-test (онлайн/оффлайн). Может быть автоматический pre-check (антифрод, анализ видео).
8. **Выдача сертификата.** После успешного прохождения в профиль пользователя добавляется «Certified Skill» с метаданными: issuer, date, level, evidence (hash/ссылка) и — опционально — Verifiable Credential (W3C-совместимый).
9. **Ревок/рецертификат.** Сертификат имеет срок (например, 12–24 мес) → по истечении требуется реаттестация или демонстрация поддерживающей активности.
# UX / UI-потоки и тексты (предложения)
* Кнопка запуска: «Play (Private) — попробовать без следов».
* При попытке сохранить: «Сохранить эту сессию? Это поместит результаты в личные черновики. Ничего не появится в профиле до подачи на сертификацию.»
* При подаче на сертификацию: «Вы собираетесь подтвердить навык. Мы передадим ваше видео и метрики компании-верификатору. Согласны?» (чекбокс).
* После сертификации: badge в профиле — «Barista (Certified) — issuer: [Company], date».
# Техническая модель данных (ключевые сущности — кратко)
* **session_ephemeral_id** (UUID, TTL) — записи активности в Private Play, уничтожаются по таймауту.
* **artifact** — {id, type(video|audio|json), owner:session_ephemeral_id|user_id, visibility:private|draft|submitted}
* **cert_request** — {user_id, sim_id, artifacts[], peer_scores[], status:pending/approved/rejected, consent_log_hash}
* **certificate** — {user_id, sim_id, issuer_id, level, issued_at, expires_at, vc_hash}
# Антифрод / честность верификации
* **Hashing артефактов**: все загружаемые артефакты подписываются хешем и хранится метаданные (fingerprint) — позволяет доказать, что документ не подделан после выдачи.
* **Поведенческий анализ**: при подаче на сертификацию собирается минимальная телеметрия (по согласию) — скорость кликов, паттерны мыши, звук/видео — для выявления аномалий.
* **Live-test**: для ключевых certs — реальный тест с наблюдателем/модератором или proctoring.
* **Peer review + company review**: комбинация разных источников оценки.
# Примеры критериев сертификации для «кофейни — бариста»
Минимальные правила (пример):
* **Количество успешных прохождений**: ≥3 (успех = время ≤ target и recipe match ≥90%).
* **Артефакты**: 2 видео процесса + заполненный чек-лист (hygiene, order accuracy) + 1 self-reflection (100–200 символов).
* **Peer review**: avg score ≥ 4/5 от не менее 3 reviewers (включая 1 от сотрудников компании/креатора симуляции).
* **Final test**: live demo (онлайн) — приготовить 3 напитка под наблюдением (оценка company).
# Вид и отображение в профиле после сертификации
* Сертификат отображается в отдельном блоке «Certified Skills» с:
* название навыка, issuer, дата выдачи, уровень (basic/intermediate/advanced), evidence (ссылка/хеш), expiry.
* При клике — раскрывается краткий отчёт: metрики, peer scores, что включает.
* Сертификат нельзя удалить без прохождения revoke flow (и логируется).
# Права и контроль пользователя
* Пользователь в любой момент может: удалить черновики, отозвать consent, запросить удаление персональных данных (в рамках GDPR/локальных правил).
* Сертифицированный навык можно отозвать (revoke) при нарушении правил — но лог о ревокации виден в профиле (transparency).
# Потенциальные бизнес/социальные эффекты
* Отсутствие автоматического трекинга повышает доверие и побуждает пробовать больше симов.
* Система «игра→сохранить→сертификация» создаёт воронку для платных сертификаций/коллабораций с компаниями.
* Verifiable Credentials повышают переносимость навыков (учебные партнеры, учебные заведения).

4. Workflow для подачи cert_request + company review panel.
5. Базовый anti-fraud pre-check + live-test integration.
6. Badge/certificate display в профиле + revocation flow.
7. Документация для компаний: требования/формат live-test & accepted artifacts.

Модуль верификации (proofs). Короткие видео/скринкасты и peer-review + блок базовой ID/anti-AI проверки.

Как платформа может защититься от злоупотреблений
Встроенные шаблоны «этичного треша»: система переписывает «агрессивный» текст в конструктивный (до публикации показывает альтернативы).
Reputation & review: оценки не только компаний, но и коллег и нейтральных ревьюеров.
Двойная верификация результатов симуляций: автоматическая + человек-модератор (для ключевых вакансий).
Прозрачные метрики: объясняемые «лайк-баллы» (как оценивались навыки).
